Why this change

The scraper was simultaneously churning duplicates on static ATS boards (Ashby, Greenhouse, Lever) and over-filtering SearXNG discovery results — starving volume while burning cache. Two failure modes, same root cause: every source ran on the same cadence with the same filters regardless of freshness characteristics.

This branch reframes sources as first-class tiers with cadence-derived rotation, LLM relevance gating on discovery, and JSON API migrations for the workhorses. A feature-flagged APScheduler drives the whole thing; a new metrics surface shows per-tier health at a glance.

The new tier model

workhorse

Static ATS boards (Ashby, Greenhouse, Lever, Workable). High duplicate rate, stable slugs. Ran full-fleet every tick before — now hash-partitioned into 4 rotation groups. Each board visited once per 24h cycle.

discovery

SearXNG web search. High noise, high variance. Now fires on alternating runs (every Nth tick) and runs through an LLM relevance gate that accepts, rejects, or keeps-uncertain-high-score before persistence.

lead

HN Hiring, RemoteOK. Already well-behaved — fires every tick, no rotation needed.

Cadence math: 6h cadence × 4 rotation groups = 24h full-cycle. Dedup TTL enforced ≥ 2× cycle (45 days) by a Pydantic validator so rotation can never outrun the dedup window.

How to operate it

Feature flags

Env varDefaultEffect
TEXTAILOR_SCRAPE_SCHEDULER0Set to 1 to enable APScheduler on dashboard startup.
ASHBY_LEGACY_HTML0Set to 1 to fall back to the old GraphQL path.
GREENHOUSE_LEGACY_HTML0Set to 1 to fall back to the 2-step HTML path.
LEVER_LEGACY_HTML0Set to 1 to restore the Playwright HTML path.

Enable the scheduler

export TEXTAILOR_SCRAPE_SCHEDULER=1
./scripts/restart-dashboard.sh

Manual tier-aware scrape

cd job-scraper
source ../venv/bin/activate
python -m job_scraper scrape -v \
    --tiers workhorse,lead \
    --rotation-group 2 \
    --run-index 14

Omit any flag to get the previous behavior (all spiders, no rotation filter).

What to expect moving forward

Less duplicate churn on ATS. Each static board now only runs every 24 hours (rotation group × 6h cadence). Dedup TTL at 45 days means you'll still catch anything that reappears.
More-targeted discovery volume. SearXNG only fires every other tick, and an LLM gate rejects obvious non-matches before they hit the dedup+filter pipeline. Rejection reasons land in rejected table with rejection_stage = 'llm_relevance'.
Playwright no longer required for Lever. Lever boards now use the public JSON API. One fewer moving part in the scrape path.
Visibility on gate health. The Ops → Metrics page shows a 7-day rollup of per-tier-per-source counters. Look for: daily net-new trending near 50/day, dedup-drop ratio dropping on workhorses, gate-overflow banner staying absent.
Scheduler is off by default. Nothing changes in production until you set TEXTAILOR_SCRAPE_SCHEDULER=1 and restart. Manual python -m job_scraper scrape runs still work unchanged.

CI/CD Security
for TexTailor

Theory-first playbook for hardening the TexTailor monorepo: Python scraper & backend, React dashboard, SearXNG container, SQLite store. Layered checks from pre-commit to release, with repo-specific rules for the parts that actually matter.

Scope: monorepo Stack: Python · TS · Docker · SQLite Posture: LAN-only dashboard

01Overview

Security in CI/CD is defense in depth. Cheap checks run first and fail loudly; expensive ones run less often. Each layer catches a different class of problem — skipping a layer leaves that class unobserved.

Guiding principle
Fail fast on cheap checks, gate merge on blocking classes, schedule expensive scans. Custom rules for this repo matter more than generic rule packs because the riskiest surface (SQL console, artifact paths, ops admin) is bespoke.

02Pipeline layers

Four stages. Checks escalate in cost and coverage.

STAGE · 01

Pre-commit

Local, sub-second. Secrets, format, lint. Catches 80% of accidents.

STAGE · 02

PR CI

Blocking gate. SAST, SCA, custom semgrep, unit tests, type check.

STAGE · 03

Main / nightly

Slow scans. SBOM, container rescan, full-history secret sweep, DAST.

STAGE · 04

Release

Sign artifacts, freeze deps, attach SBOM, tag container digest.

03Check catalog

Each category answers a different question about the change.

SAST

code patterns

Pattern-match dangerous code before it ships.

  • Python: bandit — shell=True, pickle, weak crypto
  • semgrep — FastAPI + python rulesets
  • TS: eslint-plugin-security, semgrep react rules
  • Target: injection, XSS, unsafe deserialization

SCA (dependencies)

known CVEs

Scan transitive deps for published vulnerabilities.

  • Python: pip-audit or safety
  • JS: npm audit --production, osv-scanner
  • Dependabot / Renovate auto-PRs
  • Pin with hashes (pip-compile --generate-hashes)

Secret scanning

high leverage

Detect credentials in diffs and history.

  • gitleaks / trufflehog
  • Pre-commit + PR + nightly full-history
  • Rules for OpenAI, Anthropic, Groq, Mistral, OpenRouter keys
  • Verify llm_keys.json path covered by .gitignore

Container / image

image CVEs

SearXNG image and any derived layers.

  • hadolint — Dockerfile linting
  • trivy or grype — CVE scan
  • Weekly rescan base image — new CVEs emerge
  • Flag :latest, root user, missing healthcheck

IaC / config

posture

Compose files and runtime config.

  • checkov on docker-compose.yml
  • Exposed ports review — 0.0.0.0:8899 intentional?
  • CORS allowlist, no *
  • No privileged containers

License

policy

Flag incompatible licenses before merge.

  • pip-licenses
  • license-checker (npm)
  • Fail policy: block GPL/AGPL if forbidden

SBOM + provenance

supply chain

Know what shipped so you can answer "are we affected."

  • cyclonedx-py, cyclonedx-npm
  • Attach SBOM to release artifacts
  • Pin container by digest, not tag

DAST (light)

runtime

Exercise the running app, not the source.

  • Schemathesis against FastAPI /openapi.json
  • OWASP ZAP baseline (passive in CI)
  • Active scans only in staging, never against prod

04Repo-specific rules

Generic rulesets miss the endpoints that matter here. These are custom semgrep patterns worth writing.

SELECT-only SQL console

must-have

/api/db/query must never execute non-SELECT. A regression here turns a read endpoint into DROP.

  • Semgrep rule: within routers/ops.py, block execute( without AST check
  • Unit test: POST UPDATE, DROP, INSERT → expect 400

Artifact path traversal

must-have

/api/packages/{slug}/artifact/{name} and mobile JD scan accept filenames.

  • Require .resolve().is_relative_to(base) guard
  • Reject .., absolute paths, symlinks
  • Same for /api/tailoring/runs/{slug}/artifact/{name}

Ops admin verbs

high blast radius

/ops/admin fires destructive actions with no confirm. New verbs should be reviewed by code owner.

  • CODEOWNERS on routers/ops.py, services/ops.py
  • Audit that each action writes to state_log

pdflatex shell-escape

RCE surface

tailor/compiler.py runs pdflatex. Shell-escape enabled = arbitrary command execution via malicious .tex.

  • Verify -no-shell-escape flag
  • Rule: flag any pdflatex invocation without that flag
  • Treat JD text as untrusted input

YAML loader

deserialization

config.py loads YAML. yaml.load without SafeLoader = code execution.

  • Semgrep: block yaml.load without explicit Loader=
  • Prefer yaml.safe_load

Subprocess shell=True

injection

Scraper fetchers, compiler, MLX manager all shell out.

  • Bandit B602 / B603 as errors, not warnings
  • Prefer argument lists to shell strings

String-concat SQL

injection

SQLite via db.py and services. Any f-string or + in a query = finding.

  • Semgrep: cursor.execute(f"...") or execute("..." + x)
  • Require parameterized queries (? placeholders)

Vite env leak

frontend

VITE_* env vars ship to the client bundle.

  • Rule: no secret-shaped values under VITE_
  • Keep LLM keys server-side only
Also worth a rule
dangerouslySetInnerHTML grep on frontend, target="_blank" without rel="noopener", FastAPI routes missing response_model (DB rows leak through).

05Gating policy

Not every finding blocks merge. Policy should match risk class.

New high/critical SASTBlockPR CI
Custom rule violationBlockPR CI
High CVE in direct depBlockPR CI
Transitive CVE, no fixWarnPR CI
Medium SASTWarnPR CI
License driftWarnPR CI
Container base CVETicketnightly
Full-history secret (known)Ticket + rotatenightly

Branch protection

  • Require PR review + passing CI before merge to main
  • CODEOWNERS on sensitive paths: routers/ops.py, services/llm_keys.py, services/ops.py, Dockerfile, .github/workflows/
  • No force-push to main
  • Signed commits (optional, higher bar)

06Threat model

What we are defending and what attacks are in scope. Drives priority.

  • LAN-only dashboard
    No auth currently. Assume trusted network but still harden CORS, path traversal, SQL injection, and destructive admin endpoints. A compromised LAN device becomes an admin.
  • LLM API keys
    Cloud provider keys = money. Treat secret scanning and the llm_keys.json location as first-order concerns. Rotate on any suspected leak.
  • pdflatex subprocess
    Shell-escape = arbitrary code execution. Untrusted JD content flows into .tex. Verify -no-shell-escape is set in compiler.py.
  • SearXNG container
    Faces external search providers. Keep base image patched, scan weekly, pin by digest.
  • Mobile JD OCR ingest
    Reads files from a watched folder. Validate paths, size limits, MIME. Tesseract crashes on hostile input — bound memory.
  • Prompt injection
    Malicious JD text can steer tailoring LLM outputs. Not a CI/CD check — design concern. Note it exists; don't trust LLM output as authoritative.

09Glossary

  • SAST
    Static Application Security Testing — analyze source without running it.
  • DAST
    Dynamic Application Security Testing — exercise the running app.
  • SCA
    Software Composition Analysis — check dependencies against known CVEs.
  • SBOM
    Software Bill of Materials — machine-readable inventory of what shipped.
  • CVE
    Common Vulnerabilities and Exposures — public vuln identifier.
  • IaC
    Infrastructure as Code — config files (Dockerfile, compose, Terraform).
  • Blast radius
    Scope of damage if a component is compromised.

## Part 1: How You Think Through Problems

1. **Coraline had five conflicting data sources.** Why hierarchical confidence matching? Did you try something simpler first? Was there a point where the data was messier than you expected and you had to adjust?

Coraline started as a command line tool I used locally. The problem we were trying to solve was all of our managed assets, in other words, all the assets that reported to BigFix or Jamf, needed to have vulnerability management. In other words, they needed the Rapid7 Insight VM agent. So I used the API to basically just a quick script to reconcile all three inventories. That grew into a full-stack application, just kind of out of necessity because it became apparent that there was other kind of disparate data there that we might want to take a look at. And hierarchical confidence came from basically trial and error. So IP address was almost immediately unreliable because of different network configurations. And a VPN and different IPs for different assets across different platforms. Then I tried to use MAC address only to find that MAC addresses are not super reliable. They're just kind of like the network identifier. But when you have a spread out network, it becomes a lot harder to use that as a point of reference. So ultimately we used UUID, which is something I found in my own research. It's not really a tag that anyone was keeping track of, but it is a machine identifier and it's 99.9% accurate. It's not foolproof, but it's pretty great as a reference point. If you have a machine that's reporting as a different host name in Rapid7 versus that same machine reporting in BigFix, you can use the UUID to correlate them in BigFix. You can use the UUID to correlate them. The data was incredibly messy and it remains messy. That's why CoreLine was so necessary as a full stack application hosted internally so everyone can access it and quickly align assets. Also, once we made the tracking of assets across vulnerability management, and asset management platforms trivial, or once I did that, we were able to dive deeper into other discrepancies like Rapid7's scan engine was not global across our inventory, which was problematic.

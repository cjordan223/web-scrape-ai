Remediating vulnerabilities across 10,000+ macOS and Windows devices at the UC system meant dealing with inconsistent configurations, stale agents, and compliance deadlines. I built API-driven automation using CrowdStrike RTR and Defender, but everything starts manual — I don't automate something until I've validated it by hand and put safeguards in place. The split between automated and manual isn't a design decision I made once; it's an ongoing judgment call. Zero-days, ad hoc investigations for niche users on specific toolsets — those still need human eyes. The runbooks I write for those aren't tutorials; they're "this happened, this needs to get done" — clear, concise, built for high-stress situations. I've refined them based on peer feedback, aiming for docs that actually get used under pressure.
